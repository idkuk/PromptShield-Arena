# Contributing

Thanks for helping improve PromptShield Arena!

## Quick Start
1. Create a branch from `main`.
2. Make focused, readable changes.
3. Run `openenv validate` if you change environment code.
4. Keep UI changes consistent with the design system in `server/static`.

## Coding Standards
- Prefer small, readable functions.
- Keep reward logic deterministic and bounded.
- Avoid introducing new dependencies unless necessary.

## Pull Requests
- Include a clear summary and testing notes.
- Link any related issues.

## Reporting Issues
- Describe expected vs actual behavior.
- Provide logs and steps to reproduce.
